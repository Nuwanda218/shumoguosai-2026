# 问题四可视化方案说明

问题四可视化只服务一个核心结论：

```text
问题三推荐方案已经满足问题四新增平滑约束，因此问题四最优结果与问题三一致。
```

正文主图保留 2 张：

```text
outputs/question4/constraint_strength_comparison.png
outputs/question4/q3_q4_comparison.png
```

原先的 24 小时调度曲线、逐小时变化率检验图和单独约束裕度图不再作为问题四正文主图使用。它们展示的信息可以被“约束强弱对比图”概括，继续保留会削弱问题四表达重点。

## 1. 统一绘图要求

1. 所有标题、坐标轴、图例、标注文字使用中文。
2. 使用 `visualization/README.md` 中的统一配色。
3. 背景和辅助说明放在底层，柱状图放在中间层，关键线条、标注和图例放在最高层。
4. 图例必须不透明，且不能遮挡柱状图顶部数值。
5. 不把不同量纲的原始数值强行放在同一个普通坐标轴中。

## 2. 图 1：约束强弱对比图

文件名：

```text
outputs/question4/constraint_strength_comparison.png
```

图像目的：

```text
解释为什么问题四新增平滑约束弱于主约束，因此没有改变最优解。
```

图像结构：

1. 左侧为嵌套集合交运算示意图；
2. 右侧为约束使用率柱状图。

左侧表达：

```text
Ω4 = Ω3 ∩ S_G ∩ S_R
```

其中：

```text
Ω3：问题三可行域
S_G：GPU负载变化率约束集合
S_R：传输速率变化率约束集合
```

图中需要明确画出：

```text
外层大区域：新增平滑约束范围
内层小区域：主约束可行区域
```

并标出：

```text
x3* = x4*
```

含义是问题三最优解仍位于问题四可行域中，因此集合交运算没有排除原最优点。图中只保留简单中文解释：

```text
平滑约束没有排除原最优点
```

详细的集合交运算证明放在 `问题结果md/问题四.md` 中，图中不再放过多公式。

右侧约束使用率：

| 约束 | 类型 | 使用率 |
| ---- | ---- | -----: |
| 加权误差 | 主约束 | 100.0% |
| 冷却变化 | 主约束 | 100.0% |
| GPU负载变化 | 平滑约束 | 20.8% |
| 传输速率变化 | 平滑约束 | 0.0% |

预期解释：

```text
加权误差和冷却变化已经达到约束边界，是决定调度形态的主约束；
GPU负载变化率和传输速率变化率远低于上限，是松弛约束；
因此新增平滑约束不会改变问题三最优解。
```

## 3. 图 2：问题三与问题四核心指标对比图

文件名：

```text
outputs/question4/q3_q4_comparison.png
```

图像目的：

```text
展示加入问题四新增约束后，核心指标是否发生变化。
```

绘图口径：

```text
以问题三指标为 100% 基准，将问题四指标归一化后绘制。
```

对比指标：

1. 日电费；
2. 系统能耗；
3. 总处理量；
4. 加权平均误差。

预期解释：

```text
问题四与问题三的日电费、系统能耗、总处理量和加权平均误差完全一致。
这说明问题三推荐方案已经满足问题四新增平滑约束。
```

## 4. 图像取舍说明

问题四不再保留以下正文主图：

```text
outputs/question4/schedule_profile.png
outputs/question4/change_rate_check.png
outputs/question4/constraint_margin_summary.png
```

原因：

1. `schedule_profile.png` 主要展示 24 小时调度，问题三已经充分说明；
2. `change_rate_check.png` 只说明未超限，但不能直接解释“为什么不起作用”；
3. `constraint_margin_summary.png` 只展示平滑约束裕度，没有和主约束对比。

新的 `constraint_strength_comparison.png` 同时包含集合关系和约束强弱对比，更适合论文汇报。

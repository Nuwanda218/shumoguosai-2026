# 问题四可视化方案说明

## 1. 可视化目标

问题四的关键结论不是得到一套全新的调度曲线，而是验证问题三推荐方案是否已经满足设备平滑运行要求。

问题四计算结果为：

```text
最小日电费 = 158.6374 元
总处理量 = 1.2690
加权平均误差 = 5.0000%
最大 GPU 负载变化 = 1.6667%
最大传输速率变化 = 0.0000 Mbps
最大冷却功率变化 = 0.2000 kW
```

新增约束为：

```text
GPU 负载变化不超过 8%
传输速率变化不超过 300 Mbps
```

因此图像需要回答四个问题：

1. 问题四的 24 小时调度是否平滑？
2. 新增 GPU 负载变化率约束是否起作用？
3. 新增传输速率变化率约束是否起作用？
4. 为什么问题四结果与问题三结果相同？

## 2. 文件结构设计

绘图代码建议单独放在：

```text
visualization/question4_plots.py
```

不要把可视化逻辑写入：

```text
question/question4.py
```

输出目录：

```text
outputs/question4/
  schedule_profile.png
  change_rate_check.png
  constraint_margin_summary.png
  q3_q4_comparison.png
```

问题四只需要 4 张主图。电池充放电、冷却功率和购电功率已经在问题三中充分展示，问题四不建议重复绘制完整电池图和功率图，除非论文排版需要补充。

## 3. 统一样式要求

所有图像沿用 `visualization/README.md` 中的统一配色：

```text
#8074C8, #7895C1, #A8CBDF, #D6EFF4, #F2FAFC,
#992224, #B54764, #E3625D, #EF8B67, #F0C284,
#F5EBAE, #F7FBC9
```

所有图像必须满足：

1. 标题、坐标轴、图例、标注文字全部使用中文；
2. 图像保存为 PNG；
3. 图像分辨率沿用 `visualization/style.py` 中的全局设置；
4. 图例背景不透明；
5. 图例放在最上层；
6. 不同量纲不要强行挤在同一个坐标轴；
7. 若必须使用双 Y 轴，必须确保折线图不会被柱状图遮挡。

## 4. 图层顺序原则

问题四图像必须统一采用以下图层顺序：

```text
最底层：峰、平、谷时段背景色和时段分界线
中间层：柱状图、填充区域、约束区间
最高数据层：折线图、散点图、关键点标注
最上层：图例
```

这是之前绘图中已经暴露过的问题：如果柱状图或双 Y 轴覆盖折线，会导致趋势看不清；如果时段色块盖住数据层，会削弱图像解释力。

## 5. 图 1：问题四 24 小时调度曲线

文件名：

```text
outputs/question4/schedule_profile.png
```

图像含义：

```text
展示问题四下 GPU 负载 G_t 和数据传输速率 R_t 的 24 小时调度结果。
```

坐标轴：

```text
X轴：小时 t（1-24）
左Y轴：GPU负载 G_t（%）
右Y轴：数据传输速率 R_t（Mbps）
```

绘制内容：

1. GPU 负载折线；
2. 数据传输速率折线；
3. 峰、平、谷时段背景色；
4. 时段分界线；
5. 图例：GPU负载、数据传输速率、峰时段、平时段、谷时段。

推荐图例位置：

```text
左下角
```

预期解释：

```text
GPU负载曲线已经比较平滑，最大相邻小时变化只有1.6667%，远低于8%上限。
数据传输速率始终保持1200 Mbps，所以传输速率变化率为0。
这说明问题四新增的两个约束没有改变问题三调度方案。
```

注意事项：

1. 数据传输速率曲线在 1200 Mbps 处是一条水平线，右 Y 轴范围不宜过宽；
2. 峰平谷时段背景必须放在底层；
3. 折线必须位于最高数据层。

## 6. 图 2：新增变化率约束逐小时检验图

文件名：

```text
outputs/question4/change_rate_check.png
```

图像含义：

```text
逐小时检查 GPU 负载变化、传输速率变化和冷却功率变化是否超过限制。
```

推荐绘图方式：

```text
三行子图，共享 X 轴。
```

三个子图分别为：

1. GPU 负载相邻小时变化 `|G_{t+1}-G_t|`；
2. 传输速率相邻小时变化 `|R_{t+1}-R_t|`；
3. 冷却功率相邻小时变化 `|C_{t+1}-C_t|`。

坐标轴：

```text
X轴：时段转换（1→2, 2→3, ..., 24→1）
第1行Y轴：GPU负载变化（%）
第2行Y轴：传输速率变化（Mbps）
第3行Y轴：冷却功率变化（kW）
```

绘制内容：

1. 三组变化率柱状图；
2. GPU 负载变化上限 `8%` 水平线；
3. 传输速率变化上限 `300 Mbps` 水平线；
4. 冷却功率变化上限 `0.2 kW` 水平线；
5. 第 `24→1` 个日周期边界单独标注。

推荐图例位置：

```text
每个子图右侧图外。
```

预期解释：

```text
GPU负载变化率远低于8%，传输速率变化率为0，冷却功率变化率达到0.2 kW。
这说明问题四新增约束不绑定，冷却惯性约束才是限制调度平滑性的主因。
```

注意事项：

1. 三个量纲不同，不要画在同一个 Y 轴；
2. 必须包含 `24→1` 的日周期边界；
3. 约束上限线要清晰，但不要遮挡柱状图；
4. 柱状图为中间层，约束线为最高数据层。

## 7. 图 3：约束裕度汇总图

文件名：

```text
outputs/question4/constraint_margin_summary.png
```

图像含义：

```text
用归一化比例比较三个变化率约束的紧张程度。
```

归一化指标：

```text
约束使用率 = 实际最大变化 / 约束上限
```

计算结果：

| 约束 | 实际最大变化 | 上限 | 使用率 |
| ---- | -----------: | ---: | -----: |
| GPU负载变化 | 1.6667% | 8% | 20.83% |
| 传输速率变化 | 0 Mbps | 300 Mbps | 0.00% |
| 冷却功率变化 | 0.2 kW | 0.2 kW | 100.00% |

坐标轴：

```text
X轴：约束类型
Y轴：约束使用率（%）
```

绘制内容：

1. 三个柱状图；
2. `100%` 水平线；
3. 柱状图顶部标注具体使用率；
4. 可在冷却功率柱上标注“绑定”。

推荐图例位置：

```text
右侧图外。
```

预期解释：

```text
GPU负载变化约束只使用约20.83%，传输速率变化约束完全没有使用，冷却变化约束达到100%。
问题四新增约束没有造成额外费用，主要是因为问题三冷却惯性已经使调度曲线平滑。
```

注意事项：

1. 柱状图必须标注数值；
2. 不需要画复杂曲线；
3. 该图应作为问题四结论图，放在论文中较靠后的位置。

## 8. 图 4：问题三与问题四核心指标对比图

文件名：

```text
outputs/question4/q3_q4_comparison.png
```

图像含义：

```text
展示问题四加入变化率约束后，核心经济指标是否发生变化。
```

推荐绘图方式：

```text
单张归一化分组柱状图。
```

归一化口径：

```text
问题三各指标作为 100% 基准，问题四指标除以问题三指标后绘制。
```

这样可以避免把元、kWh、处理量和百分比直接放到同一个普通坐标轴中。

绘制内容：

1. 问题三指标；
2. 问题四指标；
3. 每个柱子顶部标注原始数值；
4. 关键文字标注：“问题三方案已满足问题四新增约束”。

推荐图例位置：

```text
右侧图外，避开柱状图和顶部数值标注。
```

预期解释：

```text
问题三与问题四的日电费、处理量和误差完全一致。
问题四没有降低费用，也没有提高费用；它的作用是验证原调度方案满足设备稳定性要求。
```

注意事项：

1. 不要把元、百分比、Mbps、kW 全部放在一个普通柱状图里；
2. 若必须并列多个指标，应使用归一化或分子图；
3. 图中应明确显示“费用不变”这一结论。

## 9. 不建议作为主图的内容

以下图像不建议作为问题四主图：

1. 完整电池充放电图；
2. 完整 SOC 曲线；
3. 系统功率、电网购电功率与小时电费图；
4. 冷却稳态需求与实际冷却功率图。

原因是这些内容已经在问题三中解释过，问题四没有改变电池调度和冷却曲线。如果重复放入，会削弱问题四重点。

若论文排版需要补充，可以把这些图作为附图或复用问题三图像，但问题四正文应优先展示新增约束检验结果。

## 10. 图像表达重点

问题四图像最终要表达的核心结论是：

```text
传输速率变化率约束没有起作用，因为 R_t 已经长期取 1200 Mbps；
GPU 负载变化率约束也没有起作用，因为冷却惯性已经使负载曲线足够平滑；
真正起作用的是冷却功率变化率约束；
因此问题四结果与问题三一致，说明问题三方案已经满足长期稳定运行要求。
```

这比单纯画出一张“调度曲线”更重要。图像应围绕“约束是否绑定”组织，而不是重复展示所有变量。

# 问题一可视化方案说明

## 1. 目标

问题一的可视化不只是展示最优结果，还要解释为什么线性模型会得到：

```text
GPU 负载 G = 85%
数据传输速率 R = 1200 Mbps
误差率 E = 5%
一天总能耗 = 158.64 kWh
```

因此图像需要同时回答三个问题：

1. 误差率如何随 GPU 负载和传输速率变化？
2. 总功耗如何随 GPU 负载和传输速率变化？
3. 为什么模型倾向于提高传输速率，而不是继续提高 GPU 负载？

## 2. 文件结构设计

```text
visualization/
  README.md
  问题一图像说明.md
  style.py              # 全局中文字体、颜色、线型、图片尺寸设置
  question1_plots.py    # 问题一四张图的绘制函数
```

输出目录：

```text
outputs/question1/
  error_surface_3d.png
  power_surface_3d.png
  feasible_region_2d.png
  power_breakdown_curve.png
```

## 3. 统一样式设计

所有图像的文字都使用中文，包括：

```text
标题
坐标轴
图例
颜色条
标注文字
```

样式集中放在：

```text
visualization/style.py
```

后续如果要统一调整颜色、字体、图片大小，只改这个文件。

### 颜色建议

| 元素 | 建议颜色 | 含义 |
|---|---|---|
| 最优点 | 红色 | 关键结果 |
| 误差可行区域 | 绿色系 | 满足误差约束 |
| 不可行区域 | 浅灰色 | 不满足约束 |
| 任务量边界 | 蓝色 | 处理量约束 |
| 误差边界 | 橙色 | 误差 5% 约束 |
| 总功耗曲面 | 蓝绿渐变 | 功耗随变量变化 |
| 误差曲面 | 黄红渐变 | 误差高低变化 |

### 字体建议

优先使用系统中常见中文字体：

```text
Microsoft YaHei
SimHei
SimSun
```

并设置：

```text
axes.unicode_minus = False
```

避免负号显示异常。

## 4. 图 1：误差率三维高度图

文件名：

```text
outputs/question1/error_surface_3d.png
```

图像含义：

```text
展示误差率 E(G,R) 随 GPU 负载 G 和数据传输速率 R 的变化。
```

坐标轴：

```text
X轴：GPU负载 G（%）
Y轴：数据传输速率 R（Mbps）
Z轴：误差率 E（%）
```

绘制内容：

1. 误差率三维曲面。
2. 半透明的 `E = 5%` 约束平面。
3. `E = 5%` 约束边界线。
4. 固定 GPU 负载切片线：`G=70%`、`G=85%`、`G=100%`。
5. 最高误差角点 `(G=60, R=800, E=30)`。
6. 最低误差角点 `(G=100, R=1200, E=2)`。
7. 最优点 `(G=85, R=1200, E=5)`。
8. 中文标注“最优点”。

预期解释：

```text
GPU负载越高，误差率越低；传输速率越高，误差率越低。
最优点刚好位于 E=5% 的约束边界上。
固定 GPU 负载切片线用于直接展示：在同一 GPU 负载下，R 从 800 Mbps 提高到 1200 Mbps 时，误差率明显下降。
最高误差和最低误差角点用于说明图中已经覆盖完整变量范围，而不是只展示约束附近的局部区域。
```

注意：

```text
三维图中横纵轴的实际数值范围不同，绘图时需要设置合适的 box_aspect 和刻度，
避免数据传输速率轴比例视觉失真。
误差模型是线性平面，单独看 3D 曲面可能不够明显，因此不使用底部等高线投影，
改用固定 GPU 负载切片线、5%约束平面和高低角点标注共同展示全域变化。
```

## 5. 图 2：总功耗三维高度图

文件名：

```text
outputs/question1/power_surface_3d.png
```

图像含义：

```text
展示系统总功率 P(G,R) 随 GPU 负载 G 和数据传输速率 R 的变化。
```

坐标轴：

```text
X轴：GPU负载 G（%）
Y轴：数据传输速率 R（Mbps）
Z轴：系统总功率 P（kW）
```

系统总功率：

```text
P(G,R) = P_gpu(G) + P_trans(R) + P_cool(G)
```

绘制内容：

1. 总功率三维曲面。
2. 最优点 `(G=85, R=1200, P=6.61)`。
3. 中文标注“最优点”。

预期解释：

```text
沿 GPU 负载方向，总功率上升更快。
这是因为 GPU 负载增加会同时提高 GPU 功耗和冷却功耗。
沿传输速率方向，总功率上升较慢。
因此模型更倾向于提高传输速率来降低误差，而不是继续提高 GPU 负载。
```

## 6. 图 3：二维可行域与总功率等高线图

文件名：

```text
outputs/question1/feasible_region_2d.png
```

图像含义：

```text
展示任务量约束、误差约束、变量边界和可行域内总功率水平。
```

坐标轴：

```text
X轴：GPU负载 G（%）
Y轴：数据传输速率 R（Mbps）
```

绘制内容：

1. 任务量边界：

```text
(G / 80) * (R / 1000) = 1
```

2. 误差边界：

```text
4G + R = 1540
```

3. 不可行区域浅灰色遮罩。
4. 可行区域用总功率色块填色。
5. 最优点 `(85, 1200)`。

预期解释：

```text
最优点位于传输速率上限和误差边界的交汇附近。
任务量约束在该点不是最紧约束，误差约束才是决定最优点位置的关键。
可行域内颜色展示总功率大小，用来说明最优点是可行域中低功率边界点。
功率色块不标注具体数字，只体现相对高低，避免画面过乱。
```

## 7. 图 4：功耗分解与传输速率敏感性图

文件名：

```text
outputs/question1/power_breakdown_curve.png
```

图像含义：

```text
左图固定 R=1200 Mbps，观察 GPU 负载变化时，各部分功率和误差率如何变化。
右图固定若干 GPU 负载，观察传输速率变化时误差率如何变化。
```

坐标轴：

```text
左图X轴：GPU负载 G（%）
左图左Y轴：功率（kW）
左图右Y轴：误差率（%）
右图X轴：数据传输速率 R（Mbps）
右图左Y轴：误差率（%）
```

绘制内容：

左图：

1. GPU功率曲线。
2. 冷却功率曲线。
3. 数据传输功率曲线。
4. 系统总功率曲线。
5. 误差率曲线。
6. `E=5%` 水平线。
7. 最优点 `G=85` 的竖线。

右图：

1. `G=70%`、`G=85%`、`G=100%` 下误差随传输速率变化曲线。
2. `E=5%` 水平线。
3. 最优传输速率 `R=1200` 的竖线。
4. 对 `G=85%` 情况增加文字标注，说明 `R` 从 800 到 1200 时误差由 25% 降至 5%。
5. 用文字说明传输功率只从 0.25 kW 增至 0.41 kW，避免次坐标轴干扰误差趋势。

预期解释：

```text
传输功率在 R=1200 时仍然较小；
GPU负载提高会同时推高 GPU功率和冷却功率；
G=85 时误差刚好达到 5%，继续提高 GPU 会增加功耗但没有必要。
在固定GPU负载时，提高传输速率会明显降低误差，但传输功率增加幅度很小。
```

## 8. 可视化代码接口

建议在 `visualization/question1_plots.py` 中提供：

```text
generate_question1_grid()
plot_error_surface_3d(result, output_dir)
plot_power_surface_3d(result, output_dir)
plot_feasible_region_2d(result, output_dir)
plot_power_breakdown_curve(result, output_dir)
generate_all_question1_plots(output_dir)
```

其中：

```text
result
```

来自：

```text
question.question1.solve_question1()
```

绘图函数可以复用 `models/` 计算功耗和误差，但不要把绘图代码写入 `question/question1.py`。

# 问题二可视化方案说明

## 1. 目标

问题二的可视化需要解释分时电价下的动态调度结果：

```text
谷时段：G = 100.000%，R = 1200 Mbps
平时段：G = 78.642%，R = 1200 Mbps
峰时段：G = 60.000%，R = 1200 Mbps
最小日电费 = 162.336 元
问题一固定方案日电费 = 190.368 元
加权平均误差 = 5.0000%
```

图像需要回答五个问题：

1. 分时电价是否真的改变了 GPU 负载调度？
2. 峰、平、谷时段的系统功率和小时电费如何变化？
3. 为什么峰时段瞬时误差可以高于 5%，但全天加权误差仍满足约束？
4. 问题二相比问题一固定方案节省了多少电费？
5. 为什么数据传输速率 `R_t` 一直取上限 `1200 Mbps`？

## 2. 文件结构设计

```text
visualization/
  README.md
  问题一图像说明.md
  问题二图像说明.md
  style.py              # 全局中文字体、颜色、线型、图片尺寸设置
  question1_plots.py    # 问题一图像生成
  question2_plots.py    # 问题二图像生成，后续编写
```

输出目录：

```text
outputs/question2/
  tariff_schedule.png
  power_cost_schedule.png
  work_error_contribution.png
  q1_q2_comparison.png
  rate_max_reason.png
```

## 3. 统一样式设计

所有图像沿用 `visualization/README.md` 中给出的统一配色：

```text
#8074C8, #7895C1, #A8CBDF, #D6EFF4, #F2FAFC,
#992224, #B54764, #E3625D, #EF8B67, #F0C284,
#F5EBAE, #F7FBC9
```

绘图要求：

1. 所有标题、坐标轴、图例、颜色条、标注文字使用中文。
2. 图像保存到 `outputs/question2/`。
3. 不在求解脚本 `question/question2.py` 中混入绘图逻辑。
4. 图像优先服务论文解释，不做复杂交互图。
5. 同类图像尽量复用 `visualization/style.py` 中的字体和颜色设置。

## 4. 图 1：24 小时电价与调度曲线

文件名：

```text
outputs/question2/tariff_schedule.png
```

图像含义：

```text
展示分时电价下 GPU 负载和数据传输速率的 24 小时调度结果。
```

坐标轴：

```text
X轴：小时 t（1-24）
左Y轴：GPU负载 G_t（%）
右Y轴：数据传输速率 R_t（Mbps）
```

绘制内容：

1. GPU 负载折线或阶梯线。
2. 数据传输速率折线或阶梯线。
3. 峰、平、谷电价时段背景色。
4. 图例标注“GPU负载”“数据传输速率”“峰时段”“平时段”“谷时段”。

预期解释：

```text
GPU 负载随电价明显变化：
谷时段负载最高，峰时段负载最低，平时段处于中间水平。
这说明问题二不是简单沿用问题一固定方案，而是实现了分时电价下的动态调度。
数据传输速率始终保持 1200 Mbps，需要结合图 5 单独解释。
```


图像含义：

```text
展示系统功率 P_t 和小时电费 price_t * P_t 随时间的变化。
```

坐标轴：

```text
X轴：小时 t（1-24）
左Y轴：系统功率 P_t（kW）
右Y轴：小时电费（元）
```

绘制内容：

1. 系统功率折线或阶梯线。
2. 小时电费柱状图。
3. 峰、平、谷时段背景色。
4. 可选标注：峰时段低功率、谷时段高功率。

预期解释：

```text
谷时段电价低，系统允许较高功率运行；
峰时段电价高，系统主动降低功率；
平时段承担中间负载。
该图直接体现“低价多算、高价少算”的电费优化策略。
```

## 6. 图 3：处理量与误差贡献图

文件名：

```text
outputs/question2/work_error_contribution.png
```

图像含义：

```text
解释为什么问题二采用全天处理量加权平均误差，而不是逐小时瞬时误差。
```

坐标轴：

```text
X轴：小时 t（1-24）
左Y轴：每小时处理量 q_t
右Y轴：瞬时误差 E_t（%）
```

绘制内容：

1. 每小时处理量 `q_t` 柱状图。
2. 瞬时误差 `E_t` 折线图。
3. `E = 5%` 水平线。
4. 加权平均误差 `E_bar = 5%` 标注。
5. 峰、平、谷时段背景色。

预期解释：

```text
峰时段负载低，瞬时误差可能高于 5%，但该时段处理量也较小；
谷时段处理量较大，且误差较低；
因此全天处理量加权平均误差仍能控制在 5%。
该图用于说明加权平均误差约束比逐小时误差约束更符合“全天任务整体精度”的建模口径。
```

```text
对比问题一固定方案和问题二动态调度方案在电费、能耗、处理量和误差方面的差异。
```

坐标轴：

```text

```

绘制内容：

1. 日电费对比柱状图：

```text

问题二动态方案：162.336 元
```

2. 日总能耗对比柱状图。
3. 总处理量对比柱状图。
4. 加权平均误差对比柱状图。
5. 标注电费节省：

```text
节省 28.032 元，节省率约 14.72%
```

预期解释：

```text
问题二通过把更多任务转移到低价时段，有效降低了日电费。
问题二并不追求最低总能耗，而是追求电价加权后的最低成本。
因此动态方案的评价重点是日电费下降，而不是单纯能耗最小。
```

## 8. 图 5：传输速率取上限原因图

文件名：

```text
outputs/question2/rate_max_reason.png
```

图像含义：

```text
解释为什么问题二最优解中所有时段的数据传输速率 R_t 都等于 1200 Mbps。
```

坐标轴：

```text
X轴：数据传输速率 R（Mbps）
左Y轴：数据传输功率 P_trans（kW）
右Y轴：误差率 E（%）
```

绘制内容：

1. 数据传输功率曲线：

```text
P_trans(R) = 0.25 + 0.0004(R - 800)
```

2. 固定 GPU 负载下的误差率曲线：

```text
G = 60%
G = 78.642%
G = 100%
```

3. `R = 1200 Mbps` 竖线。
4. 文字标注：

```text
R 从 800 到 1200，传输功率仅增加 0.16 kW。
R 从 800 到 1200，误差率下降 20 个百分点。
```

预期解释：

```text
提高传输速率的功率代价很小，但可以显著降低误差并提高处理量。
相比之下，提高 GPU 负载会同时增加 GPU 功率和冷却功率。
因此模型优先把传输速率推到上限，再通过 GPU 负载调节适应峰、平、谷电价。
这就是问题二中 R_t 始终取 1200 Mbps 的原因。
```

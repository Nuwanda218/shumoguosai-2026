# 问题二设计说明

## 1. 问题定位

问题二在问题一的基础上加入分时电价，目标由“最小化一天总能耗”改为“最小化一天总电费”。

问题一采用静态方案：

```text
G_t = G
R_t = R
```

问题二必须允许调度量随电价变化，否则分时电价只会把问题一的固定功率乘以不同电价，无法体现“低价多运行、高价少运行”的动态调度思想。

因此问题二采用动态调度：

```text
G_t：第 t 小时 GPU 负载
R_t：第 t 小时数据传输速率
```

## 2. 分时电价

一天 24 小时按题目分为三类：

```text
峰时段：9:00-11:00, 18:00-20:00，电价 2.0 元/kWh，共 4 小时
平时段：6:00-9:00, 11:00-18:00, 20:00-22:00，电价 1.2 元/kWh，共 12 小时
谷时段：22:00-6:00，电价 0.8 元/kWh，共 8 小时
```

由于同一电价类别内没有额外区别，正式求解时采用峰、平、谷三类统一调度：

```text
谷时段：G_valley, R_valley
平时段：G_flat, R_flat
峰时段：G_peak, R_peak
```

再把三类结果展开成 24 小时调度表。

这样做的好处是：

1. 变量少，求解稳定。
2. 结果容易解释。
3. 符合题目给出的电价分段结构。
4. 后续问题三、问题四可以在此基础上再引入电池、冷却惯性和变化率约束。

## 3. 处理量模型

第 `t` 小时处理量为：

```text
q_t = (G_t / 80) * (R_t / 1000) / 24
```

全天任务量约束为：

```text
sum(q_t) >= 1
```

## 4. 误差模型

继续采用问题一确定的 8-GPU 叠加口径：

```text
E_t = 30 - 0.2(G_t - 60) + 0.05(800 - R_t)
```

问题二不采用逐小时瞬时误差约束：

```text
E_t <= 5
```

原因是该约束会把每个小时都压到接近同一个最低功耗可行点：

```text
G = 85
R = 1200
```

这样分时电价无法产生真正的动态调度。

因此问题二采用全天处理量加权平均误差约束：

```text
sum(q_t * E_t) / sum(q_t) <= 5
```

等价写法：

```text
sum(q_t * E_t) <= 5 * sum(q_t)
```

该约束的含义是：不同小时完成的处理量不同，对最终分析结果的误差贡献也不同，因此误差应按处理量加权。

## 5. 功率模型

第 `t` 小时系统功率为：

```text
P_t = P_gpu(G_t) + P_trans(R_t) + P_cool(G_t)
```

其中三个分量全部复用 `models/` 中的基础公式：

```text
P_gpu(G_t)：8 块 GPU 总功率
P_trans(R_t)：数据传输功率
P_cool(G_t)：冷却稳态功率
```

问题二还不考虑电池，也不考虑冷却惯性，因此冷却功率仍按稳态需求直接计算。

## 6. 目标函数

问题二目标为最小化一天总电费：

```text
min Cost = sum(price_t * P_t)
```

其中每小时长度为 1 小时，所以：

```text
price_t * P_t
```

数值上就是该小时电费。

## 7. 约束条件

完整模型为：

```text
min sum(price_t * P_t)
```

满足：

```text
60 <= G_t <= 100
800 <= R_t <= 1200
sum(q_t) >= 1
sum(q_t * E_t) / sum(q_t) <= 5
```

采用峰、平、谷三类变量后，仍然通过展开成 24 小时数组来计算总处理量、加权平均误差和日电费。

## 8. 预期调度特征

合理结果应具有以下趋势：

```text
谷时段：电价最低，负载和传输速率较高，承担较多处理量，并压低整体误差。
平时段：电价中等，承担补充任务。
峰时段：电价最高，负载较低，处理量较少，允许较高瞬时误差。
```

需要注意：

```text
峰时段瞬时误差可能高于 5%，这不是模型错误。
因为问题二约束的是全天处理量加权平均误差，而不是每小时误差。
```

## 9. 与问题一的对比口径

问题一结果：

```text
G = 85
R = 1200
P = 6.61 kW
```

若按问题二电价直接计算问题一固定方案的日电费：

```text
Cost_Q1 = 6.61 * sum(price_t)
```

问题二应与该固定方案进行对比：

```text
电费降低额 = Cost_Q1 - Cost_Q2
电费降低率 = (Cost_Q1 - Cost_Q2) / Cost_Q1
```

该对比用于说明分时电价动态调度的价值。

## 10. 本阶段输出

本阶段只实现问题二求解代码，不实现图像与结果文件导出。

计划输出：

1. 峰、平、谷三类调度结果。
2. 24 小时 `G_t, R_t, P_t, price_t, cost_t` 表。
3. 日总电费。
4. 日总能耗。
5. 总处理量。
6. 加权平均误差。
7. 与问题一固定方案的电费对比。

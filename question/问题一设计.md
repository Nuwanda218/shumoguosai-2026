# 问题一求解设计讨论

## 1. 目标

问题一要求在满足实验室基本运行约束、总处理量约束和误差约束的前提下，设计一套 AI 算力功耗降低方案，使一天总能耗最小。

当前阶段只讨论设计，不写代码。

## 2. 已确认建模口径

### 决策变量

问题一采用静态运行方案，即一天 24 小时使用同一组参数：

```text
G：每块 GPU 负载，单位 %
R：数据传输速率，单位 Mbps
```

后续代码中会把静态变量扩展成 24 小时数组：

```text
G_t = G, t = 0,1,...,23
R_t = R, t = 0,1,...,23
```

这样可以直接复用 `models.task.total_work()` 和 `models.task.weighted_average_error()`。

### 变量范围

```text
60 <= G <= 100
800 <= R <= 1200
```

### 处理量约束

处理量公式：

```text
q_t = (G_t / 80) * (R_t / 1000) / 24
```

问题一要求：

```text
sum(q_t) >= 1
```

静态方案下等价于：

```text
(G / 80) * (R / 1000) >= 1
```

### 误差约束

采用 8 块 GPU 共同降低误差的模型：

```text
E = 30 - 0.025 * 8 * (G - 60) + 0.05 * (800 - R)
```

即：

```text
E = 30 - 0.2 * (G - 60) + 0.05 * (800 - R)
```

问题一为静态方案，日均加权误差等于单小时误差，因此约束为：

```text
E <= 5
```

可整理为：

```text
4G + R >= 1540
```

### 目标函数

一天总能耗：

```text
min 24 * [P_gpu(G) + P_trans(R) + P_cool(G)]
```

其中：

```text
P_gpu(G) = 8 * 单块 GPU 功率 / 1000
P_trans(R) = 0.25 + 0.0004 * (R - 800)
P_cool(G) = 1.2 + 0.12 * (G - 60)
```

## 3. 预期求解结果

根据当前模型，最优点应接近：

```text
G = 85%
R = 1200 Mbps
日总能耗 = 158.64 kWh
误差 = 5%
总处理量 = 1.275
```

说明：

- `G=85, R=1200` 刚好满足误差边界 `E=5`。
- 处理量大于 1，表示该方案完成量超过一天需求。
- 若后续认为“不允许超额处理”，需要把处理量约束从 `>=1` 改成 `=1` 或增加超额处理惩罚。这个点需要讨论后再决定。

### 方案设计说明

问题一不是动态调度问题，而是寻找一组固定运行参数，使实验室在全天连续运行时满足约束并降低功耗。因此它适合作为后续问题二、三、四的基准模型。

求解方案分为四步：

1. 用 `G` 表示每块 GPU 的同步负载，用 `R` 表示数据传输速率。
2. 根据题目参考数据建立三类功耗函数：GPU 功耗、数据传输功耗、冷却功耗。
3. 建立两个必要约束：总处理量不少于日需求量，数据分析误差不超过 5%。
4. 在可行域内最小化一天总能耗。

这个设计的特点是结构清晰、约束少、可解释性强。它不追求调度灵活性，而是回答一个基础问题：如果全天采用稳定运行策略，最低能耗是多少。

### 可行域解释

任务量约束和误差约束共同决定可行域。

任务量约束：

```text
(G / 80) * (R / 1000) >= 1
```

它表示 GPU 负载和传输速率可以互相补偿。例如 GPU 负载较高时，传输速率可以适当降低；传输速率较高时，GPU 负载可以适当降低。

误差约束：

```text
4G + R >= 1540
```

它要求 GPU 负载和传输速率共同提供足够的分析精度。由于传输速率对误差的改善系数较大，并且传输能耗增长较小，误差约束会强烈推动传输速率向上。

最终最优点落在误差约束边界上：

```text
G = 85, R = 1200, E = 5
```

这说明问题一的最优方案不是随意取到上限，而是在“传输速率已达到上限”的条件下，选择刚好满足误差要求的最低 GPU 负载。

### 结果指标解释

最优解的各项指标含义如下：

| 指标 | 数值 | 解释 |
|---|---:|---|
| GPU 负载 | 85% | 在 R=1200 时，满足误差 5% 所需的最低 GPU 负载 |
| 数据传输速率 | 1200 Mbps | 传输速率达到题目上限，原因是其降误差成本很低 |
| GPU 集群功率 | 2.00 kW | 8 块 GPU 在 85% 负载下的总功率 |
| 数据传输功率 | 0.41 kW | 相比 GPU 和冷却功率较小 |
| 冷却功率 | 4.20 kW | 随 GPU 负载上升，是总功耗主要来源之一 |
| 系统总功率 | 6.61 kW | 三部分功率之和 |
| 一天总能耗 | 158.64 kWh | `6.61 * 24` |
| 总处理量 | 1.275 | 超过日需求量 27.5% |
| 数据分析误差 | 5.00% | 刚好达到误差约束边界 |

这个结果中最值得解释的是：传输功率只有 `0.41 kW`，远小于冷却功率和 GPU 功率，因此模型倾向于使用高传输速率来换取更低的 GPU 负载和更低的误差。

## 4. 结果合理性分析

当前问题一的计算结果是数学模型下的最优解，但不应直接理解为“工程上一定最合理”。核心原因是：题目给出的线性关系会让提高数据传输速率变成一种非常便宜的降误差手段。

在当前误差模型下：

```text
E = 30 - 0.2 * (G - 60) + 0.05 * (800 - R)
```

误差约束 `E <= 5` 等价于：

```text
4G + R >= 1540
```

因此：

```text
R = 1200 时，G >= 85
G = 100 时，R >= 1140
G = 60 时，R >= 1300，不可行
```

这说明，在该模型中，若要满足 5% 误差，传输速率必须处于较高水平；并不是 `900-1060 Mbps` 就能保证误差达标。若出现这种区间，说明使用了不同的误差公式、不同的符号解释或不同的系数假设。

任务量约束同样不是两个独立区间，而是耦合关系：

```text
(G / 80) * (R / 1000) >= 1
```

等价于：

```text
R >= 80000 / G
```

例如：

```text
G = 66.7, R = 1200  可满足任务量
G = 100,  R = 800   可满足任务量
G = 66.7, R = 800   不满足任务量
```

当前模型之所以倾向于把 `R` 取到 1200，是因为传输速率的边际能耗代价远小于 GPU 负载提升的边际能耗代价。

传输速率每增加 `1 Mbps`：

```text
功率增加 0.0004 kW
误差降低 0.05%
```

GPU 负载每增加 `1%`：

```text
GPU 与冷却功率合计约增加 0.136-0.144 kW
误差降低 0.2%
```

换算为“降低 1% 误差所需增加的功率”：

```text
通过提高传输速率：约 0.008 kW
通过提高 GPU 负载：约 0.68-0.72 kW
```

两者相差几十倍。因此，优化器会优先把传输速率推到上限，再尽量降低 GPU 负载，得到 `G=85, R=1200`。

这个结果符合当前题目线性模型，但工程解释上需要补充说明：它依赖“传输速率能耗线性增长且增长很小”这一假设。如果认为长期使用最高传输速率不符合实际，可以在后续扩展模型中加入：

1. 高传输速率非线性能耗，例如超过 1000 Mbps 后功耗增长更快。
2. 长期高传输速率稳定性惩罚。
3. 推荐运行区间，例如对 `R=1200` 施加软惩罚。
4. 敏感性分析，比较 `R=1000, 1100, 1200` 下所需 GPU 负载和总能耗。

因此，问题一代码先实现“题目线性模型基准解”；论文或说明中再解释该结果的模型依赖性，并可在后续加入改进模型对比。

### 与直觉不一致的原因

从工程直觉看，长期把传输速率设置到最高值可能显得激进。但数学模型只会根据题目给出的代价和收益进行权衡，不会自动考虑网络拥塞、链路稳定性、设备老化或长期满载风险。

当前题目参数隐含了两个强假设：

1. 传输速率从 800 Mbps 提高到 1200 Mbps，功率只从 0.25 kW 增加到 0.41 kW。
2. 传输速率提高对误差改善非常明显，每提高 1 Mbps，误差降低 0.05%。

在这两个假设下，传输速率是最便宜的精度改善手段。因此优化器选择 `R=1200` 是模型内部合理的结果。

如果要让结果更接近工程经验，需要显式把工程经验写入模型，例如：

```text
高传输速率额外惩罚
传输速率超过 1000 Mbps 后功耗非线性增加
传输速率长期接近上限时增加稳定性风险项
限制推荐运行区间而不是只使用硬件极限
```

否则，单纯使用题目线性参数时，模型没有理由避免 `R=1200`。

### 需要在论文中表达的结论

问题一的结论建议这样表述：

```text
在题目给出的线性功耗和误差关系下，最优节能方案为 GPU 负载 85%、数据传输速率 1200 Mbps。该方案使误差刚好达到 5% 上限，日能耗为 158.64 kWh。由于数据传输速率提升带来的功耗增量远小于 GPU 负载和冷却功率增量，模型倾向于优先提高传输速率来降低误差，再降低 GPU 负载以节省能耗。因此该结果是题目线性模型下的基准最优解。
```

同时需要补充：

```text
若考虑长期高传输速率运行的稳定性风险或非线性能耗，最优点可能向较低传输速率、更高 GPU 负载方向移动。该问题可作为模型敏感性分析或改进模型进一步讨论。
```

## 7. 可视化需求

问题一至少需要两类图：

1. 功率构成柱状图：GPU、传输、冷却三部分功率。
2. 结果汇总图或表：最优 `G`、`R`、能耗、误差、处理量。

这些图不在 `question/question1.py` 中直接画，而是调用 `visualization/`。

### 为解释结果合理性需要增加的图

为了说明 `G=85, R=1200` 不是偶然结果，可视化不应只画最终值，还要展示功耗、误差和约束如何随 `G`、`R` 变化。

建议至少设计以下图像。

#### 图 1：能耗曲面或等高线图

横轴：

```text
GPU 负载 G
```

纵轴：

```text
数据传输速率 R
```

颜色：

```text
一天总能耗
```

图中叠加：

```text
任务量约束边界
误差约束边界
最优点 (85, 1200)
```

用途：

```text
展示可行域内能耗如何变化，并说明最优点位于误差约束边界和传输速率上限交汇附近。
```

#### 图 2：误差率等高线图

横轴为 `G`，纵轴为 `R`，颜色表示误差率 `E`。

重点标出：

```text
E = 5% 约束线
E < 5% 可行区域
E > 5% 不可行区域
最优点
```

用途：

```text
说明如果传输速率不够高，即使 GPU 负载较高，也可能无法满足误差要求；同时说明 R=1200 时 G=85 正好压在误差边界上。
```

#### 图 3：固定传输速率下的功耗与误差曲线

选取几组传输速率：

```text
R = 1000, 1100, 1200
```

横轴为 `G`，左轴画系统总功率，右轴画误差率。

用途：

```text
比较不同传输速率下，为达到 5% 误差需要多少 GPU 负载，以及对应功率如何变化。
```

这张图可以直观解释为什么 `R=1200` 会胜出：在最高传输速率下，达到误差要求所需的 GPU 负载最低，总功率也最低。

#### 图 4：边际代价对比图

画两根柱子：

```text
通过提高传输速率降低 1% 误差所需功率增量
通过提高 GPU 负载降低 1% 误差所需功率增量
```

当前估算：

```text
传输速率：约 0.008 kW / 1%误差
GPU 负载：约 0.68-0.72 kW / 1%误差
```

用途：

```text
直接解释为什么模型优先提高传输速率，而不是继续提高 GPU 负载。
```

### 推荐优先级

第一阶段可先做三张图：

1. 能耗等高线图，展示最优点和可行域。
2. 误差率等高线图，展示 5% 误差边界。
3. 固定传输速率下的功耗与误差曲线，解释为什么 R=1200。

边际代价对比图可以作为论文补充图，或在文字分析中用表格表达。

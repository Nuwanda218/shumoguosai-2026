# 问题四设计：考虑负载与传输速率平滑约束的长期调度模型

## 1. 问题定位

问题四在问题三模型基础上，进一步考虑设备长期稳定运行要求。

题目给出的新增要求是：

1. GPU 负载相邻小时变化幅度不超过 `8%`；
2. 数据传输速率相邻小时变化幅度不超过 `300 Mbps`。

因此，问题四不重新建立功耗、处理量、误差、电池和冷却模型，而是在问题三“电池储能 + 冷却惯性”的动态调度模型上，增加 GPU 负载和传输速率的变化率约束。

问题四的目标仍然是最小化一天总电费。

## 2. 与前三问的继承关系

问题一确定了基础功耗、处理量和误差模型。

问题二在问题一基础上加入分时电价，并采用全天处理量加权平均误差约束，使动态调度可以在低价时段承担更多任务、高价时段降低工作强度。

问题三在问题二基础上加入电池储能系统和冷却系统惯性，得到长期可重复运行的日调度方案。

问题四继续继承问题三的全部设定，只新增设备平滑约束：

```text
|G_{t+1} - G_t| <= 8
|R_{t+1} - R_t| <= 300
```

这里 `G_t` 表示第 `t` 小时每块 GPU 的负载百分比，8 块 GPU 同步调节；`R_t` 表示第 `t` 小时的数据传输速率。

## 3. 建模口径修正

### 3.1 误差约束仍采用处理量加权平均

问题四不能把误差约束写成简单平均，也不能写成传输速率的平均。应继续采用问题二、问题三已经确定的口径：

```math
\frac{\sum_{t=1}^{24} q_t E_t}{\sum_{t=1}^{24} q_t} \le 5
```

其中：

```math
q_t=\frac{G_t}{80}\cdot \frac{R_t}{1000}\cdot \frac{1}{24}
```

```math
E_t=30-0.2(G_t-60)+0.05(800-R_t)
```

该约束表示全天数据分析误差不超过 `5%`，并且高处理量小时对总误差影响更大。

### 3.2 冷却惯性限制的是实际冷却功率

问题四仍应继承问题三的冷却惯性模型。冷却系统的稳态需求为：

```math
C_t^*=1.2+0.12(G_t-60)
```

但实际冷却功率是决策变量 `C_t`，应满足：

```math
C_t \ge C_t^*
```

```math
|C_{t+1}-C_t| \le 0.2
```

不能写成：

```text
|C_{t+1}^* - C_t^*| <= 0.2
```

因为这样会把冷却系统惯性错误地变成 GPU 负载变化限制。题目第四问已经单独给出了 GPU 负载变化率约束，因此两类约束应同时保留，但不能混写。

### 3.3 电池采用长期循环口径

题目只要求终止 SOC 不低于 `20%`，但如果不设置每日循环条件，模型会把第一天初始电量当作一次性免费资源。

问题三已经采用长期重复运行口径，因此问题四继续使用循环保留电量：

```math
S_1=S_{25}=S_{\text{res}}
```

问题三推荐方案选择：

```math
S_{\text{res}}=24\text{ kWh}
```

问题四主模型建议固定使用该值，便于与问题三结果直接对比。若后续需要做稳健性分析，可继续对 `S_res = 8, 12, 16, 20, 24, 32 kWh` 做梯度对比。

## 4. 符号定义

| 符号 | 含义 | 单位 |
| --- | --- | --- |
| `t` | 小时编号，`t=1,2,...,24` | h |
| `G_t` | 第 `t` 小时每块 GPU 负载 | % |
| `R_t` | 第 `t` 小时数据传输速率 | Mbps |
| `C_t` | 第 `t` 小时实际冷却功率 | kW |
| `C_t^*` | 第 `t` 小时冷却稳态需求 | kW |
| `P_t^ch` | 第 `t` 小时电池充电功率 | kW |
| `P_t^dis` | 第 `t` 小时电池放电功率 | kW |
| `S_t` | 第 `t` 小时开始时电池电量 | kWh |
| `c_t` | 第 `t` 小时分时电价 | 元/kWh |
| `q_t` | 第 `t` 小时完成的日任务量比例 | 无量纲 |
| `E_t` | 第 `t` 小时数据分析误差 | % |
| `P_t^grid` | 第 `t` 小时从电网购电功率 | kW |

## 5. 基础模型

### 5.1 GPU 功耗

单块 GPU 功率采用题目给定的分段线性模型：

```math
P_g(G_t)=
\begin{cases}
180+3(G_t-60), & 60\le G_t\le 80 \\
240+2(G_t-80), & 80<G_t\le 100
\end{cases}
```

8 块 GPU 同步调节，因此 GPU 集群功率为：

```math
P_t^{gpu}=\frac{8P_g(G_t)}{1000}
```

### 5.2 数据传输功耗

传输速率为 `800 Mbps` 时功率为 `0.25 kW`，每增加 `200 Mbps` 功率增加 `0.08 kW`，因此：

```math
P_t^{trans}=0.25+0.0004(R_t-800)
```

### 5.3 冷却功率

冷却稳态需求为：

```math
C_t^*=1.2+0.12(G_t-60)
```

问题四继续使用实际冷却功率 `C_t`：

```math
C_t \ge C_t^*
```

### 5.4 电网购电功率

```math
P_t^{grid}=P_t^{gpu}+P_t^{trans}+C_t+P_t^{ch}-P_t^{dis}
```

为避免向电网反送电，加入：

```math
P_t^{grid}\ge 0
```

## 6. 目标函数

问题四目标为最小化全天总电费：

```math
\min F=\sum_{t=1}^{24} c_t P_t^{grid}
```

其中 `c_t` 为分时电价：

| 时段类型 | 电价 |
| --- | --- |
| 峰时段 | 2.0 元/kWh |
| 平时段 | 1.2 元/kWh |
| 谷时段 | 0.8 元/kWh |

## 7. 约束条件

### 7.1 变量边界

```math
60\le G_t\le 100
```

```math
800\le R_t\le 1200
```

```math
1.2\le C_t\le 6.0
```

```math
0\le P_t^{ch}\le 12
```

```math
0\le P_t^{dis}\le 12
```

```math
8\le S_t\le 40
```

### 7.2 任务量约束

```math
\sum_{t=1}^{24} q_t\ge 1
```

其中：

```math
q_t=\frac{G_t}{80}\cdot \frac{R_t}{1000}\cdot \frac{1}{24}
```

### 7.3 误差约束

```math
\frac{\sum_{t=1}^{24} q_t E_t}{\sum_{t=1}^{24} q_t}\le 5
```

其中：

```math
E_t=30-0.2(G_t-60)+0.05(800-R_t)
```

### 7.4 电池 SOC 递推

电池容量为 `40 kWh`，充放电效率为 `90%`。用 `S_t` 表示电池电量，单位为 `kWh`：

```math
S_{t+1}=S_t+0.9P_t^{ch}-\frac{P_t^{dis}}{0.9}
```

```math
t=1,2,\dots,24
```

其中 `S_25` 表示下一天开始时的电量。

长期循环约束为：

```math
S_1=S_{25}=S_{\text{res}}
```

主模型取：

```math
S_{\text{res}}=24
```

### 7.5 电池不能同时充放电

```math
P_t^{ch}P_t^{dis}=0
```

求解时可采用互斥约束、罚函数或充放电状态变量处理。若沿用前三问的连续优化实现，可继续用非线性约束限制同一小时充放电重叠。

### 7.6 冷却惯性约束

实际冷却功率必须满足稳态需求：

```math
C_t\ge C_t^*
```

相邻小时实际冷却功率变化不超过 `0.2 kW`：

```math
|C_{t+1}-C_t|\le 0.2,\quad t=1,2,\dots,23
```

为保证长期日调度可重复，加入日周期边界：

```math
|C_1-C_{24}|\le 0.2
```

### 7.7 GPU 负载变化率约束

题目要求每块 GPU 从当前小时到下一小时的负载变化幅度不超过 `8%`。由于 8 块 GPU 同步调节，只需要对统一负载变量 `G_t` 施加约束：

```math
|G_{t+1}-G_t|\le 8,\quad t=1,2,\dots,23
```

为了保证每天循环运行时第 24 小时到下一天第 1 小时也不出现突变，加入：

```math
|G_1-G_{24}|\le 8
```

### 7.8 传输速率变化率约束

题目要求数据传输速率从当前小时到下一小时变化幅度不超过 `300 Mbps`：

```math
|R_{t+1}-R_t|\le 300,\quad t=1,2,\dots,23
```

同样加入日周期边界：

```math
|R_1-R_{24}|\le 300
```

## 8. 问题四完整优化模型

综上，问题四可写为：

```math
\min \sum_{t=1}^{24} c_t
\left[
\frac{8P_g(G_t)}{1000}
 + P_t^{trans}
 + C_t
 + P_t^{ch}
 - P_t^{dis}
\right]
```

约束为：

```math
60\le G_t\le 100,\quad 800\le R_t\le 1200
```

```math
q_t=\frac{G_t}{80}\cdot \frac{R_t}{1000}\cdot \frac{1}{24},\quad \sum_{t=1}^{24}q_t\ge 1
```

```math
E_t=30-0.2(G_t-60)+0.05(800-R_t)
```

```math
\frac{\sum_{t=1}^{24}q_tE_t}{\sum_{t=1}^{24}q_t}\le 5
```

```math
C_t^*=1.2+0.12(G_t-60),\quad C_t\ge C_t^*
```

```math
|C_{t+1}-C_t|\le 0.2,\quad |C_1-C_{24}|\le 0.2
```

```math
S_{t+1}=S_t+0.9P_t^{ch}-\frac{P_t^{dis}}{0.9}
```

```math
8\le S_t\le 40,\quad S_1=S_{25}=24
```

```math
0\le P_t^{ch}\le 12,\quad 0\le P_t^{dis}\le 12,\quad P_t^{ch}P_t^{dis}=0
```

```math
P_t^{grid}\ge 0
```

```math
|G_{t+1}-G_t|\le 8,\quad |G_1-G_{24}|\le 8
```

```math
|R_{t+1}-R_t|\le 300,\quad |R_1-R_{24}|\le 300
```

## 9. 与问题三结果的预期关系

问题四新增的是工程平滑约束，因此最优费用通常不会低于问题三。若问题三结果本身已经比较平滑，则问题四结果可能与问题三非常接近。

需要重点观察三类变化：

1. GPU 负载曲线是否被进一步平滑；
2. 电池是否仍能在峰时段有效放电；
3. 冷却功率是否仍是紧约束。

传输速率在前几问中通常倾向于取 `1200 Mbps`。如果问题四结果仍然保持 `R_t=1200 Mbps`，则传输速率变化率约束不绑定。这是合理结果，说明传输速率上限策略在功耗代价较低、降误差效果明显的条件下仍具有优势。

## 10. 求解建议

问题四是带有非线性目标和非线性约束的 24 小时连续优化问题。建议沿用问题三的求解框架：

1. 决策变量仍为 `G_t, R_t, C_t, P_t^ch, P_t^dis`；
2. SOC 由充放电功率递推得到；
3. 在问题三约束列表中新增 `G_t` 和 `R_t` 的相邻小时变化率约束；
4. 初始解可直接使用问题三推荐方案，再通过平滑处理使其满足新增约束；
5. 主模型固定 `S_res=24 kWh`，后续如有需要再做循环保留电量敏感性分析。

求解后应输出：

1. 最小日电费；
2. 24 小时 GPU 负载和传输速率；
3. 电池充放电和 SOC；
4. 实际冷却功率；
5. 加权平均误差和总处理量；
6. 最大 GPU 负载变化、最大传输速率变化和最大冷却功率变化；
7. 与问题三结果的费用和曲线对比。

## 11. 可行性与检验指标

结果需要至少检验以下条件：

| 检验内容 | 判定标准 |
| --- | --- |
| 任务量 | `sum(q_t) >= 1` |
| 误差 | `sum(q_t E_t) / sum(q_t) <= 5` |
| SOC | `8 <= S_t <= 40` |
| 循环运行 | `S_1 = S_25 = 24` |
| 冷却需求 | `C_t >= C_t^*` |
| 冷却变化 | `max |C_{t+1}-C_t| <= 0.2`，含日周期边界 |
| GPU 负载变化 | `max |G_{t+1}-G_t| <= 8`，含日周期边界 |
| 传输速率变化 | `max |R_{t+1}-R_t| <= 300`，含日周期边界 |
| 购电功率 | `P_t^grid >= 0` |

若上述条件全部满足，即可认为问题四模型与题目要求一致，并且可以与问题三进行直接对比。
